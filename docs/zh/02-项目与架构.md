## 项目简介

**obvector4j** 是 OceanBase 向量存储的 Java JDBC SDK，基于标准 JDBC 连接 OceanBase，提供：

- **向量表管理**：建表、建向量索引、插入 / 删除 / ANN 检索
- **混合搜索（Hybrid Search）**：全文 + 向量 RRF 融合、标量 Filter + 向量检索
- **JSON 虚拟表**：将 JSON 存储映射为可 DDL 操作的逻辑表（`ObVecJsonClient`）

适用场景：RAG 检索、商品/文档推荐、带业务过滤条件的语义搜索等。

## 架构总览

```
                    ┌─────────────────────────────────────┐
                    │  应用层（你的 Java 业务代码）          │
                    └──────────────────┬──────────────────┘
                                       │
              ┌────────────────────────┴────────────────────────┐
              │                                                 │
    ┌─────────▼─────────┐                           ┌───────────▼──────────┐
    │   ObVecClient     │                           │   ObVecJsonClient    │
    │   （向量存储入口）  │                           │   （JSON 虚拟表入口）  │
    └─────────┬─────────┘                           └───────────┬──────────┘
              │                                                 │
    ┌─────────┼─────────────────────────────────────────────────┤
    │         │         │         │         │                   │
    ▼         ▼         ▼         ▼         ▼                   ▼
 schema    model    hybrid    filter   version              json_table
    │         │         │         │         │                   │
    └─────────┴─────────┴────┬────┴─────────┘                   │
                             ▼                                  │
                           util ◄────────────────────────────────┘
                             │
                             ▼
                    OceanBase JDBC Driver
                             │
                             ▼
                      OceanBase 集群
```

**设计原则**

| 原则 | 说明 |
|------|------|
| 薄入口 | 根包仅暴露 `ObVecClient` / `ObVecJsonClient`，其余按职责分子包 |
| 版本自适应 | OceanBase ≥ 4.6.0 走原生 `HYBRID_SEARCH` SQL；低版本走兼容 SQL + 客户端 RRF |
| 类型安全 | Filter DSL、Schema 建模、Sqlizable 值类型，减少手写 SQL 出错 |

## 包路径说明

根路径：`com.oceanbase.obvector4j`

### 根包（入口）

| 类 | 路径 | 功能 |
|----|------|------|
| `ObVecClient` | `com.oceanbase.obvector4j` | **主入口**。连接管理、Collection CRUD、向量 ANN 查询、混合搜索 Builder、`getOceanBaseVersion()` |
| `ObVecJsonClient` | `com.oceanbase.obvector4j` | **JSON 虚拟表入口**。继承 `ObVecClient`，解析 CREATE/ALTER/DROP 等 DDL 并映射到底层 JSON 元数据表 |

**典型用法**

```java
ObVecClient client = new ObVecClient(uri, user, password);

// 混合搜索（推荐：链式 Builder，无需单独 import Builder 类）
client.textVectorSearch().table("docs").textQuery("...").search();
client.scalarVectorSearch().table("products").filter(filter).search();

// 版本能力探测
client.supportsHybridSearchSql();  // true 表示支持 HYBRID_SEARCH 原生 SQL
```

### `schema` — 表结构与索引

路径：`com.oceanbase.obvector4j.schema`

| 类 | 功能 |
|----|------|
| `DataType` | 列类型枚举（INT32、FLOAT_VECTOR、JSON 等），含转 MySQL/OceanBase DDL 的 `Convert2MySQL()` |
| `ObCollectionSchema` | 向量「Collection」整体 schema，聚合字段与索引 |
| `ObFieldSchema` | 单字段定义（主键、自增、维度、nullable 等） |
| `IndexParam` / `IndexParams` | HNSW 向量索引参数（distance、ef_search、lib 等） |
| `Visitable` | Schema 访问者基类，内部用于生成 CREATE TABLE DDL |

**何时使用**：调用 `ObVecClient.createCollection()` 建表时。

```java
import com.oceanbase.obvector4j.schema.*;
```

---

### `model` — JDBC 行值类型

路径：`com.oceanbase.obvector4j.model`

| 类 | 功能 |
|----|------|
| `Sqlizable` | 可绑定到 `PreparedStatement` 的抽象值类型 |
| `SqlVector` | 向量字面量（float 数组 → `[1.0, 2.0, ...]`） |
| `SqlText` / `SqlInteger` / `SqlDouble` / `SqlFloat` | 标量值包装 |
| `SqlizableFactory` | 从 `ResultSet` 按 `DataType` 构造查询结果（内部使用） |

**何时使用**：插入数据、处理 `query()` / 混合搜索返回的 `HashMap<String, Sqlizable>`。

```java
import com.oceanbase.obvector4j.model.*;
```

---

### `hybrid.core` — HYBRID_SEARCH DSL（4.6.0+）

路径：`com.oceanbase.obvector4j.hybrid.core`（**低于 4.6.0 调用 `hybridSearch()` 即报错**）

| 类 | 功能 |
|----|------|
| `HybridSearch` | 4.6.0+ 入口（`ObVecClient.hybridSearch()`） |
| `HybridSearchSupport` | 版本门控 |
| `HybridSearchCustomBuilder` / `HybridSearchDsl` | 自定义 DSL 拼装与执行 |
| `hybrid.core.dsl.*` | `HybridDslKeys`、`HybridDslNode`、`HybridDsl` 关键字封装 |

### `hybrid` — 混合搜索（版本自适应）

路径：`com.oceanbase.obvector4j.hybrid`

| 类 | 可见性 | 功能 |
|----|--------|------|
| `HybridTextVectorSearchBuilder` | **公开** | 全文 + 向量（自动选原生/兼容路径） |
| `HybridScalarVectorSearchBuilder` | **公开** | 标量 + 向量 |
| `HybridSearchEngine` | 内部 | 执行引擎 |
| `HybridResultMerger` | 内部 | 兼容模式客户端 RRF |

```
supportsHybridSearchSql() == true  (OceanBase >= 4.6.0)
    └─► HYBRID_SEARCH(table, '{...json dsl...}')  单条 SQL

supportsHybridSearchSql() == false (< 4.6.0)
    └─► 分别执行向量检索 + 全文检索 + 客户端 RRF 合并
```

详细用法见 [03-混合搜索](03-混合搜索.md)；**DSL 语法与示例**见 [05-HYBRID_SEARCH-DSL语法](05-HYBRID_SEARCH-DSL语法.md)。

### `filter` — 类型安全过滤条件

路径：`com.oceanbase.obvector4j.filter`

| 类 | 功能 |
|----|------|
| `Filter` | 过滤条件 AST（等于、范围、AND/OR/NOT、IN 等） |
| `FilterBuilder` | 流式构建 Filter（**推荐对外使用**） |
| `FilterMapper` | Filter → HYBRID_SEARCH JSON DSL 的 filter 子句 |
| `FilterSqlConverter` | Filter → SQL WHERE（兼容模式使用） |

```java
import com.oceanbase.obvector4j.filter.Filter;
import com.oceanbase.obvector4j.filter.FilterBuilder;

Filter f = FilterBuilder.and(
    FilterBuilder.key("category_id").isEqualTo(1),
    FilterBuilder.key("price").isGreaterThanOrEqualTo(100.0)
);
```

详细用法见 [04-Filter过滤](04-Filter过滤.md)。

---

### `version` — 版本检测

路径：`com.oceanbase.obvector4j.version`

| 类 | 功能 |
|----|------|
| `OceanBaseVersion` | 解析 `OB_VERSION()` / JDBC 版本串，比较语义版本，常量 `HYBRID_SEARCH_SQL_MIN = 4.6.0` |

通常通过 `ObVecClient.getOceanBaseVersion()` 间接使用，无需直接 import。

---

### `json_table` — JSON 虚拟表实现

路径：`com.oceanbase.obvector4j.json_table`

| 类 | 功能 |
|----|------|
| `JsonTableMetadata` | 用户维度下的 JSON 表/列元数据 |
| `JsonTableColumn` | 列类型解析与映射 |
| `JsonData` / `JsonInt` / `JsonVarchar` 等 | JSON 字段值的类型化表示 |

**说明**：该包为 `ObVecJsonClient` 内部实现，一般业务代码不需要直接 import。

---

### `util` — 工具类

路径：`com.oceanbase.obvector4j.util`

| 类 | 功能 |
|----|------|
| `VectorMetric` | 向量距离函数名校验（l2 / cosine / inner_product）与向量字面量格式化 |

内部模块共用，对外通常通过 Builder 的 `.metric("l2")` 间接使用。

## 包依赖关系

```
filter ──┐
util  ───┼──► hybrid ──► ObVecClient
schema ──┤
model ───┘

json_table ──► ObVecJsonClient ──► ObVecClient
version    ──► ObVecClient
```

无循环依赖：底层模块（filter / util / schema / model / version）不依赖 hybrid 或 Client。

## 公开 API 速查

| 需求 | 推荐 import |
|------|-------------|
| 连接与所有高层操作 | `com.oceanbase.obvector4j.ObVecClient` |
| JSON 虚拟表 | `com.oceanbase.obvector4j.ObVecJsonClient` |
| 建表 schema | `com.oceanbase.obvector4j.schema.*` |
| 插入/结果值类型 | `com.oceanbase.obvector4j.model.*` |

## 版本要求摘要

| 能力 | OceanBase 版本 |
|------|----------------|
| 向量 ANN 检索、基础 CRUD | 4.x（视集群向量特性而定） |
| 原生 `HYBRID_SEARCH` SQL | **≥ 4.6.0** |
| 低于 4.6.0 | SDK 自动降级为兼容 SQL + 客户端 RRF |

运行时以 `client.supportsHybridSearchSql()` 为准，勿仅依赖 JDBC 驱动版本号字符串。
