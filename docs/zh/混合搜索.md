## 概述

OceanBase **4.6.0** 起提供 `HYBRID_SEARCH` SQL，可在一条 `SELECT` 中组合：

- 全文检索（`match` / `multi_match`）
- 向量检索（`knn`）
- 标量过滤（`filter`）
- 数据库端 RRF 融合排序

obvector4j 检测到版本 **≥ 4.6.0** 时自动走原生 `HYBRID_SEARCH`；低于 4.6.0 时回退兼容模式（客户端 RRF 或 `APPROXIMATE LIMIT` 向量 SQL）。

---

## 版本要求

| 能力 | 最低 OceanBase 版本 | SDK 行为 |
|------|---------------------|----------|
| `HYBRID_SEARCH` 原生 SQL | **4.6.0** | 自动启用 |
| 全文 + 向量（兼容） | 4.4.x+ | &lt; 4.6.0 时使用 |
| 标量 + 向量（兼容） | 4.3.x+ | &lt; 4.6.0 时使用 |

### 版本检测

```java
ObVecClient client = new ObVecClient(uri, user, password);

OceanBaseVersion version = client.getOceanBaseVersion();
System.out.println("OceanBase version: " + version);

if (client.supportsHybridSearchSql()) {
    System.out.println("走 HYBRID_SEARCH 原生 SQL");
} else {
    System.out.println("走兼容模式");
}
```

## 快速开始

Builder API 不变，SDK 按版本自动选路径。

### 标量 + 向量（带 Filter）

```java
import com.oceanbase.obvector4j.ObVecClient;
import com.oceanbase.obvector4j.filter.Filter;
import com.oceanbase.obvector4j.filter.FilterBuilder;

ObVecClient ob = new ObVecClient(uri, user, password);

Filter filter = FilterBuilder.and(
    FilterBuilder.and(
        FilterBuilder.key("category_id").isEqualTo(1),
        FilterBuilder.key("price").isGreaterThanOrEqualTo(50.0)),
    FilterBuilder.key("price").isLessThanOrEqualTo(250.0));

ArrayList<HashMap<String, Sqlizable>> results = ob.scalarVectorSearch()
    .table("products")
    .queryVector(queryVector)
    .filter(filter)
    .topk(5)
    .outputFields("price", "category_id")
    .search();
```

### 全文 + 向量（RRF 融合）

```java
ArrayList<HashMap<String, Sqlizable>> results = ob.textVectorSearch()
    .table("documents")
    .queryVector(queryVector)
    .textFields("title", "content")
    .textQuery("OceanBase 数据库")
    .metric("cosine")
    .topk(10)
    .rankWindowSize(20)
    .outputFields("title", "content")
    .search();
```

Filter 构建详见 [Filter过滤](Filter过滤.md)。

## 自定义 SQL 与 DSL 入口

> **DSL 语法规则、bool/knn/rank 详解与完整示例** → [dsl语法](dsl语法.md)

### 自定义 SQL

```java
// DDL / DML
client.executeSql("UPDATE products SET status = 1 WHERE id = 100");

// 任意 SELECT，列类型自动推断
ArrayList<HashMap<String, Sqlizable>> rows = client.querySql(
    "SELECT id, name, price FROM products WHERE category_id = 1 LIMIT 10");
```

### 自定义 HYBRID_SEARCH DSL（4.6.0+，低于 4.6.0 报错）

须通过 **`client.hybridSearch()`** 进入（连接版本 &lt; 4.6.0 时立即抛出 `UnsupportedOperationException`）：

```java
import com.oceanbase.obvector4j.hybrid.core.HybridSearch;
import com.oceanbase.obvector4j.hybrid.core.dsl.HybridDsl;
import com.oceanbase.obvector4j.hybrid.core.dsl.HybridDslKeys;
import com.oceanbase.obvector4j.hybrid.core.dsl.HybridDslNode;

// 入口：版本 < 4.6.0 直接报错
HybridSearch hs = client.hybridSearch();

hs.customSearch()
    .table("documents")
    .query(HybridDsl.bool()
        .must(HybridDsl.match("content", "OceanBase"))
        .filter(HybridDsl.term("status", 1)))
    .knn(HybridDsl.knn("embedding", queryVector, 10))
    .rank(HybridDsl.rrf(10, 60))
    .size(5)
    .outputFields("id", "title", "content")
    .search();
```

`customHybridSearch()` 等价于 `hybridSearch().customSearch()`，同样受版本校验。

更多 DSL 类型（`multi_match`、`query_string`、JSON/Array、`minimum_should_match` 默认值、多路 knn、WRRF 等）见 **[HYBRID_SEARCH DSL 语法](dsl语法.md)**。

## 实现路径说明

```
textVectorSearch() / scalarVectorSearch()
              │
              ▼
   supportsHybridSearchSql() ?
        /              \
      是                否
       │                 │
       ▼                 ▼
 HYBRID_SEARCH      兼容模式
 (4.6.0+ 原生)      · 全文+向量：双路查询 + 客户端 RRF
                    · 标量+向量：WHERE + APPROXIMATE LIMIT
```

| 路径 | 全文+向量 | 标量+向量 |
|------|-----------|-----------|
| **4.6.0+** | 单条 SQL，库端 RRF | `knn` + `filter` DSL |
| **&lt; 4.6.0** | MATCH + 向量 SQL，客户端 RRF | WHERE + 向量 TopK |

## 使用限制（OceanBase 侧）

1. 仅支持**堆表**（`ORGANIZATION HEAP`）
2. 向量列须建**向量索引**（HNSW 系列）
3. 全文列须建**全文索引**
4. 同层不能直接用 `WHERE` / `ORDER BY` / `LIMIT`，需子查询包装
5. 不支持 `_source`，通过 `SELECT` 列列表指定返回字段

**建表示例**：

```sql
CREATE TABLE t (...)
  ORGANIZATION HEAP WITH COLUMN GROUP(all columns);

CREATE VECTOR INDEX idx ON t(vec)
  WITH (distance=l2, type=hnsw_sq, lib=vsag);
```

## 过滤条件建议

| 方式 | 4.6.0+ | 兼容模式 |
|------|--------|----------|
| `Filter` 对象（**推荐**） | ✅ DSL 自动转换 | ✅ SQL WHERE |
| 字符串 `.filter("a = 1")` | ❌ 回退兼容模式 | ✅ |

## 常见问题

**如何确认当前走哪条路径？**

```java
ob.supportsHybridSearchSql();  // true = HYBRID_SEARCH SQL
```

**Testcontainers 能测 HYBRID_SEARCH 吗？**

`oceanbase-ce:latest` 可能 &lt; 4.6.0，测试走兼容模式仍可通过。验证原生 SQL 需连接 **4.6.0+** 实例。

**`metric()` 在 HYBRID_SEARCH 下还有效吗？**

原生路径使用索引上的 `distance` 配置；`metric()` 主要在兼容模式下选择距离函数。

## 远程集成测试

类：`com.oceanbase.obvector4j.integration.remote.HybridSearchRemoteIT`

```bash
OCEANBASE_REMOTE_IT=1 \
OCEANBASE_HOST=<your-host> \
OCEANBASE_PORT=<your-port> \
OCEANBASE_USER=<your-user> \
OCEANBASE_PASSWORD=<your-password> \
OCEANBASE_DATABASE=<your-database> \
mvn test -Premote-it
```

也可设置 `OCEANBASE_URI=jdbc:oceanbase://<host>:<port>/<database>`（仍需 `OCEANBASE_USER` / `OCEANBASE_PASSWORD`）。

| 用例 | 说明 |
|------|------|
| `testOceanBaseVersionDetected` | 版本与 HYBRID_SEARCH 能力 |
| `testPureVectorSearch` | 纯向量 TopK |
| `testScalarVectorSearchWithFilter` | 标量 + 向量 + Filter |
| `testTextVectorHybridSearch` | 全文 + 向量 RRF |
| `testHybridSearchSqlNativePath` | 4.6.0+ 原生 SQL（纯向量） |
| `testHybridSearchTextVectorRrfNativeSql` | 4.6.0+ 全文+向量 RRF |
| `testHybridSearchTextVectorWithFilterNativeSql` | 4.6.0+ 全文+向量+Filter |
| `HybridSearchDocIT` | 4.6.0+ 提测文档 DSL 全场景（见 [DSL 语法](dsl语法.md#调试与测试)） |
